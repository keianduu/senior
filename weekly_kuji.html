# 楽天シニア クジ機能（クロスユース＆有料会員化促進）PRD ## 1. 背景・目的 (Background & Objectives) * **クロスユースの促進:** 「歩く」「記録する」「読む（コラム）」「開く（ログイン）」の4大ミッションを横断的に利用してもらうことで、ユーザーの健康習慣をより強固にし、アプリのDAU・継続率およびARPUの向上を狙う。 * **有料会員（サブスクリプション）の獲得:** ミッション達成時の「クジ」という強いインセンティブを活用し、無料会員と有料会員の獲得ポイント差を提示。クジの結果画面という最もユーザーの感情が高ぶるタイミングで有料転換を促す（Loss Aversion：損失回避性を活用したアップセル）。 ## 2. ターゲットユーザー (Target Audience) * **プライマリ:** 無料で楽天シニアを利用しており、1〜2つのミッション（例：歩数のみ）しか利用していないアクティブユーザー。 * **セカンダリ:** すでに複数ミッションをこなしているが、有料会員登録には至っていないロイヤルティの高い無料ユーザー。 ## 3. UI/UX要件 (UI/UX Requirements) ### 3.1 フローティングアクションボタン (FAB) * **配置:** `index.html` 画面右下（または左下）にフローティング表示。スクロール中も常に視界に入るようにする。 * **状態表示:** * **Non-active（未達成）:** グレーアウト、または「進行中」を示すアイコン（例：歩いてるパンダと進捗ゲージ）。 * **Active（達成・引ける状態）:** ゴールドや赤などの目立つ色でアニメーション（キラキラなど）させ、「クジが引けます！」というバッジを表示。 ### 3.2 クジ条件確認ポップアップ（未達時） * **トリガー:** FABタップ時（未達状態の場合）。 * **ミッション進捗の可視化:** * 「歩こう」「健康管理」「コラム」「ログイン」の4つのミッションごとに、スタンプ（0/7〜7/7）の進捗状況をプログレスバーまたはスタンプカード形式で視覚的に表示。 * 未達のミッションには「挑戦する」等のCTAボタンを配置し、各ミッション画面へ直接遷移させる。 * **インセンティブの訴求（モチベーション維持とアップセル）:** * **工夫点:** 無料ユーザーが「少ない」とガッカリしないよう、「ハズレなし！必ず当たる！」を強調。その上で、「**プレミアム会員ならポイント最大100倍！**」といった見せ方で有料プランの魅力を提示する。 * 表記例: * 無料会員: 1等 10pt / 2等 5pt / 3等 3pt / 4等 1pt（必ずもらえる！） * 有料会員: 1等 1000pt / 2等 100pt / 3等 50pt / 4等 10pt ### 3.3 クジ実行画面（達成時） * **トリガー:** FABタップ時（条件達成状態の場合）、またはポップアップ内で全達成を確認した状態。 * **CTA:** 「特選クジを引く！」などの期待感を高める大きなボタン。 * **アニメーション:** シニア層に親しみやすいガラポン（福引）や宝箱などの演出。低スペック端末を考慮し、CSSアニメーションまたは軽量なLottie/SVGを活用。 ### 3.4 結果発表・有料転換画面 * **当選結果表示:** 「おめでとうございます！○等当選！」と大きく表示。 * **有料転換CTA（マジックモーメント）:** * 無料会員が引いた場合、結果と同時に**「待って！今プレミアム会員になると、この○ptが【○○pt】にアップします！」**という見せ方で登録CTAを設置。 * 「この結果を確定して次へ（無料のまま）」というスキップ導線も明確に用意し、ダークパターン（騙し討ち）にならないようシニアに優しい誠実な設計とする。 ## 4. KPI (Key Performance Indicators) 1. **クロスユース率:** 4ミッション全てのアクティブユーザー割合（週次・月次）。 2. **クジ実行率:** ミッション達成者のうち、実際にクジを引いたユーザーの割合。 3. **CVR（有料会員転換率）:** クジ結果画面のCTAを経由した有料会員（月額300円）の新規登録数。 --- ## 👨‍💻 マイケルからの確認事項（Business & 開発目線）
